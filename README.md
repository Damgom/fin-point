# Fin-Point

## Commit Message Convention
- INIT : 새로운 branch 를 만들고 작업을 시작
- FEAT : 새로운 기능의 추가
- REFACTOR : 코드 리팩토링
- FIX : 오류 수정, 버그 FIX
- CONFIG : 설정파일 추가, 수정
- TEST : 테스트 코트, 리팩토링 테스트 코드 추가


## 깃 플로우
- main
  - dev
    - feat

> feat 네이밍 규칙 예시 : feat-oauth-kakao

> feat에서 개발 브랜치 생성후 feat에서 PR(pull request) -> 기능 테스트후 dev 브랜치로 PR
